# Техническое задание на проект: FitMacro Planner

1. Цель проекта:  
   Веб‑сервис для ведения дневника питания с автоматическим подсчётом **КБЖУ**, постановкой целей и просмотром прогресса (графики/прогноз).

2. Роли пользователей:
   - Гость: просмотр лендинга и описания сервиса
   - Пользователь: регистрация/вход, добавление еды и веса, постановка целей, просмотр аналитики
   - Администратор: управление справочником продуктов в админ‑панели Django

3. Модели данных:
   - Модель 1: `FoodItem` — название/бренд, КБЖУ на 100г, источник (manual/openfoodfacts), внешний id
   - Модель 2: `Meal` — пользователь, дата, тип приёма пищи
   - Модель 3: `MealItem` — связь с приёмом пищи и продуктом, граммовка (расчёт КБЖУ)
   - Модель 4: `Goal` — пользователь, целевые калории/макросы, даты/вес (старт/цель)
   - Модель 5: `WeightLog` — пользователь, дата, вес

4. Ключевой функционал:
   * Регистрация и вход
   * Поиск продуктов и импорт из внешнего API
   * Добавление приёма пищи (продукт + граммы) с расчётом КБЖУ
   * Добавление веса
   * Постановка целей и отображение прогресса
   * Дашборд: графики калорий/веса и прогноз

5. Внешние интеграции / Аналитика:
   - OpenFoodFacts API (поиск/импорт продуктов)
   - Pandas (агрегации по дням)
   - Plotly (графики)
